# Design Tokens

The design token system uses a three-tier architecture to separate brand intent from platform implementation.

---

## Three-Tier Architecture

```
Tier 1 — Primitive Tokens
  oklch(0.50 0.22 250)  →  --primitive-blue-50

Tier 2 — Semantic Tokens
  --primitive-blue-50   →  --color-primary-default

Tier 3 — Component Tokens
  --color-primary-default  →  button background
```

### Tier 1 — Primitive Tokens

Raw values generated by the OKLCH engine from the two brand seeds. Never reference these directly in components.

```css
--primitive-primary-050: oklch(0.95 0.05 250);
--primitive-primary-100: oklch(0.90 0.08 250);
--primitive-primary-200: oklch(0.83 0.10 250);
/* ... 11 steps per palette, 8 palettes */
```

### Tier 2 — Semantic Tokens

Intent-bearing CSS custom properties. These are what components consume:

| Token | Purpose |
|---|---|
| `--color-primary-default` | Primary action color |
| `--color-primary-hover` | Primary hover state |
| `--color-primary-active` | Primary pressed state |
| `--color-primary-foreground` | Text on primary bg |
| `--color-accent-default` | Secondary action / highlight |
| `--color-background-default` | Page background |
| `--color-background-subtle` | Subtle background (hover areas) |
| `--color-surface-card` | Card background |
| `--color-surface-popover` | Popover / overlay background |
| `--color-foreground-default` | Body text |
| `--color-foreground-muted` | Secondary text |
| `--color-foreground-subtle` | Placeholder / disabled text |
| `--color-border-default` | Default border |
| `--color-border-muted` | Subtle divider |
| `--color-success-*` | Success state (default, foreground, subtle) |
| `--color-warning-*` | Warning state |
| `--color-error-*` | Error state |
| `--color-info-*` | Info state |

### Tier 3 — Shape Tokens

| Token | Purpose |
|---|---|
| `--shape-radius-button` | Button border radius |
| `--shape-radius-card` | Card border radius |
| `--shape-radius-input` | Input border radius |
| `--shape-radius-badge` | Badge border radius |
| `--shape-radius-dialog` | Dialog border radius |
| `--shape-preset-button` | Human-readable preset name |

---

## Token Pipeline

```
BrandConfig (JSON)
   ↓
OKLCH Color Engine  →  generates 176 raw color values
   ↓
Style Dictionary v4  →  transforms to platform outputs
   ↓  ↓  ↓
CSS     JS     iOS/Android (future)
```

### Build Outputs

| Output | Path | Consumer |
|---|---|---|
| CSS custom properties | `dist/tokens.css` | Web apps via `BrandProvider` |
| ESM JS object | `dist/tokens.js` | React Native (future) |
| TypeScript types | `dist/tokens.d.ts` | Type-safe token access |

---

## Using Tokens in Components

A simple CSS example using the generated custom properties:

```css
.button-primary {
  background: var(--color-primary-default);
  color: var(--color-primary-foreground);
  border-radius: var(--shape-radius-button);
}
```

---

## Token Validation

The `@travel/tokens` package exports a validator:

```ts
import { validateBrandConfig } from '@travel/tokens/validator';

const result = validateBrandConfig({
  id: 'my-brand',
  name: 'My Brand',
  primarySeed: { l: 0.50, c: 0.22, h: 250 },
  accentSeed: { l: 0.62, c: 0.20, h: 55 },
  shape: 'rounded',
  neutralTemperature: 'neutral',
  semanticTemperature: 'neutral',
});
// result.valid: boolean, result.errors: string[]
```
